云计算实验三

学号：202200130096 姓名： 周仁发 班级： 计科22.2班
实验题目：利用云平台搭建个人博客
实验学时：2 实验日期： 2025/3/19
实验目的：本实验旨在通过实践操作，掌握基于云计算环境的网站部署技能。
具体包括：构建环境并实现个人博客/网站的搭建，撰写实验报告，并将这份报告作为一篇博文发布在自己的博客 / 网站上。

硬件环境： 
联网的计算机一台

软件环境：
Windows or Linux

实验步骤与内容：
一、环境准备

  1. 安装Node.js
    Hexo基于Node.js开发，需先安装Node.js；官网下载安装包：Node.js官网 ； 安装后验证：命令行输入 node -v 和 npm -v 显示版本号即成功。

  2. 安装Git
    Git用于版本控制和部署到GitHub/Gitee；验证安装：git --version 查看版本号。
    二、初始化Hexo项目

  3. 全局安装Hexo
    在命令行执行以下命令安装Hexo脚手架工具：
    bash
    npm install -g hexo-cli

  4. 创建博客目录并初始化
    bash
    hexo init myblog # 创建项目文件夹
    cd myblog # 进入项目目录
    npm install # 安装依赖

  5. 本地预览
    bash
    hexo g # 生成静态文件
    hexo s # 启动本地服务器
    访问 http://localhost:4000 查看默认博客页面。
    三、部署到GitHub/Gitee

  6. 创建GitHub Pages仓库
    仓库名必须为 用户名.github.io（如 alice.github.io）。
    Gitee用户需创建同名仓库并手动开启Pages服务。

  7. 配置Hexo部署设置
    修改项目根目录的 _config.yml 文件：
    deploy:
    type: git
    repo: https://github.com/zhourenfa/zhourenfa.github.io.git # 仓库地址
    branch: main # GitHub默认分支为main

  8. 安装部署插件并推送
    bash
    npm install hexo-deployer-git –save
    hexo clean # 清理缓存
    hexo g # 重新生成
    hexo d # 部署到远程仓库
    四、文章管理与发布

  9. 新建文章
    bash
    hexo new “云计算实验三” # 在source/_posts生成Markdown文件
    编写Markdown内容。

  10. 草稿与发布
    草稿：hexo new draft “标题”，发布时移动到 _posts 目录。 生成并部署：hexo g -d或分步执行hexo g+hexo d`。
    结论与分析：
    此次实验通过Node、Git、Hexo以及Github页面实现了个人网站的搭建，使我对主流云平台的使用有了深刻的理解；大大提升了编写博客的便利性和效率。
    学会使用云平台和Hexo等工具后，给我的生活带来了极大的便利。
